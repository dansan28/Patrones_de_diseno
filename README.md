# Patrones_de_diseno

# Catálogo Técnico de Patrones de Diseño de Software

Este repositorio contiene un catálogo técnico de patrones de diseño de software
utilizados en arquitecturas de software complejas. Su propósito es servir como
base de consulta para la toma de decisiones durante el diseño de sistemas,
documentando cada patrón conforme a una estructura técnica estandarizada.

El proyecto se desarrolla utilizando control de versiones mediante GitHub, lo
que permite evidenciar la colaboración activa de los integrantes del equipo a
través del historial de commits.

## Organización del Repositorio

Los patrones de diseño están organizados por categoría. Cada patrón cuenta con
una carpeta propia que contiene su documentación, implementaciones de código y
su diagrama UML correspondiente.

Estructura de Documentación por Patrón

Cada patrón de diseño documentado en este repositorio incluye obligatoriamente
los siguientes cinco elementos, de acuerdo con los lineamientos establecidos en
la actividad:

Nombre
Identificación oficial del patrón de diseño.

Categoría
Clasificación del patrón como Creacional, Estructural o de Comportamiento.

Propósito
Descripción breve que explica qué hace el patrón y por qué es útil.

Código de Referencia
Implementación funcional del patrón en al menos dos lenguajes de programación
(Java y Python).

Estructura UML
Diagrama de clases que ilustra la relación entre los objetos participantes.

Tecnologías Utilizadas

Lenguajes de programación: Java, Python

Control de versiones: Git y GitHub

Documentación: Markdown

Diagramas: UML / PlantUML
